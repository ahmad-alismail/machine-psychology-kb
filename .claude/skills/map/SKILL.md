Regenerate `wiki/crosswalk/coverage-map.md` — the at-a-glance scorecard of which
dangerous behaviors (`safety-concepts/`) have an experiment (`paradigms/`) and which
are still research gaps. This is the project's primary gap-finder, so accuracy matters
more than speed.

## Workflow

### Step 1 — Enumerate rows and columns

- **Columns = paradigms.** List every file in `wiki/paradigms/`. Read each page's
  frontmatter (`lens`) and its `## Target Behaviors` (a.k.a. "Detects")
  section.
- **Rows = safety-concepts.** List every file in `wiki/safety-concepts/`.

Do not hardcode either list — derive both fresh from disk every run, so new pages
appear automatically and deleted ones drop out.

### Step 2 — Determine each cell

For every (safety-concept, paradigm) pair, decide the mark by reading the **paradigm**
page (the paradigm is the source of truth for what it targets):

- **`✓` (targeted)** — the paradigm's `## Target Behaviors` links this safety-concept
  as a primary target it directly surfaces or measures.
- **`~` (weak/indirect)** — the paradigm bears on the behavior only indirectly: it
  probes a *prerequisite* or *substrate*, the link is hedged ("indirectly", "necessary
  ingredient", "prerequisite"), or it is flagged with a `> [!warning]` honest-mapping
  caveat. When the paradigm explicitly says it does **not** directly evidence the
  behavior, use `~`, not `✓`.
- **(blank)** — no relationship asserted on the paradigm page.

Cross-check against each safety-concept's `## Detected By` list; if the two disagree,
**trust the paradigm page** and note the mismatch in the output (it usually means a
back-link needs fixing — surface it rather than silently resolving it).

### Step 3 — Build the page

Write `wiki/crosswalk/coverage-map.md` with this structure (preserve frontmatter,
bumping `date_modified` to today):

```markdown
---
type: crosswalk
field: machine-psychology
lens: na
tags: [coverage, research-gap]
date_created: <keep original>
date_modified: <today>
---

# Coverage Map — Behaviors × Paradigms

> [!info] In plain terms
> <2–4 plain sentences: rows are dangerous behaviors, columns are experiments,
> a filled cell means we can probe that behavior, an empty row is a research gap.>

> [!warning] Current state
> <N> of <M> target behaviors have no paradigm yet. Regenerated by `/map`.

Legend: ✓ = targeted · ~ = weak/indirect · (blank) = no experiment yet

| Behavior \ Paradigm | <col headers: plain ABBR, no wikilinks> |
|---|:--:| ... |
| [[safety-concepts/...]] | ✓ | ~ | | ... |
...

## Paradigm abbreviations
<one row per column abbreviation, mapping it to the full page link>
| ABBR | Paradigm |
|---|---|
| FBT | [[paradigms/false-belief-task]] |

## Lens of each paradigm
| Paradigm | Lens |
|---|---|
| [[paradigms/...]] | <lens> |

## Biggest gaps (no paradigm yet)
<bulleted list of every safety-concept whose entire row is blank, grouped sensibly;
mention that the agentic/scheming cluster typically needs game-theoretic or
agentic-scenario designs.>
```

Conventions:
- Use a short uppercase abbreviation as each column header — plain text only
  (e.g. `FBT`, `CBP`), **never** an aliased wikilink. The full `[[paradigms/...]]`
  links live in the `## Paradigm abbreviations` legend below the matrix, one per
  abbreviation. (Aliased wikilinks — display text after a vertical bar inside the
  double brackets — are not used anywhere in this wiki.)
- Order rows so that better-covered behaviors sit near the top and fully-uncovered
  ones group at the bottom — this makes the gap block visually obvious.
- A behavior is a **gap** iff its entire row is blank (no `✓` and no `~`). Count gaps
  for the `> [!warning]` banner.

### Step 4 — Report

Print a short summary to the user: total paradigms, total behaviors, number of gaps,
and any paradigm↔safety-concept back-link mismatches found in Step 2. **Do not
auto-commit** — leave the regenerated map for the user to review (it is committed as
part of the next `ingest:` commit, or manually).
